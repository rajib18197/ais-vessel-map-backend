feat: add get-vessels-in-bounds slice (bonus geospatial endpoint)

- Add GET /api/vessels/in-bounds?swLng=&swLat=&neLng=&neLat= returning
  all active vessels within a geographic bounding box
- Validate query params with Zod: longitude clamped to [-180, 180],
  latitude to [-90, 90], coerced from query strings to numbers
- Cross-field refinements ensure swLng < neLng and swLat < neLat so
  the bounding box is geometrically valid before hitting the DB
- Query uses MongoDB $geoWithin $box operator with coordinates in
  [lng, lat] order as required by GeoJSON convention
- Filter by ACTIVE_VESSEL_WINDOW_MS so only recently seen vessels
  are returned, consistent with get-all-vessels
- Parse DB output through z.array(vesselSummarySchema) at the boundary
- Add usecase tests covering result shape, empty bounds, $box filter
  structure, active window cutoff, and correct [lng, lat] coordinate
  ordering in the query
- Register under /in-bounds in vessels.router.ts before /:mmsi to
  prevent Express param shadowing
